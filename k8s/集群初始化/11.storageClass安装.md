# 首先部署 nfs 服务器 到 node1

```
# 注意，这个nfs要在每一个节点都安装才可以，这一点在新家node节点的时候需要注意

# 安装 nfs
yum install nfs-utils -y

# 启动 nfs
systemctl start nfs-server

# 查看 nfs 版本
cat /proc/fs/nfsd/versions

# 创建共享目录
mkdir -p /data/nfs
cd /data/nfs
mkdir rw
mkdir ro

# 设置共享目录 export ， node1节点汇总操作
vim /etc/exports
/home/nfs/rw 192.168.1.0/24(rw,sync,no_subtree_check,no_root_squash)
/home/nfs/ro 192.168.1.0/24(ro,sync,no_subtree_check,no_root_squash)
/data/nfs/rw 192.168.1.0/24(rw,sync,no_subtree_check,no_root_squash)
/data/nfs/ro 192.168.1.0/24(ro,sync,no_subtree_check,no_root_squash)
# 重新加载
exportfs -f
systemctl reload nfs-server


# 到其他测试节点安装 nfs-utils 并加载测试， master节点中做操作
mkdir -p /mnt/nfs/rw
mount -t nfs 192.168.1.11:/home/nfs/rw /mnt/nfs/rw
mount -t nfs 192.168.1.11:/home/nfs/ro /mnt/nfs/ro


```

# 遇到一个问题

由于权限对 persistentvolumeclaims 权限设置不够，导致 pvc pv 无法创建。
出错系信息是
ExternalProvisioning 3m28s (x26 over 9m32s) persistentvolume-controller waiting for a volume to be created, either by external provisioner "fuseim.pri/ifs" or manually created by system administrator，

重点：要通过这个 log 命令去查询原因，才得知是权限设置不够
kubectl logs nfs-client-provisioner-79d9759549-nrvgd -n kube-system

命令来应用更新的 RBAC 定义，得知权限不足
因此在 rbac 中添加相应的权限设置，得以解决问题
