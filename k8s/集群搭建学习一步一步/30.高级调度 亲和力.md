# 介绍

RequiredDuringSchedulingIgnoredDuringExecution
硬亲和力，即支持必须部署在指定的节点上，也支持必须不部署在指定的节点上

PreferredDuringSchedulingIgnoredDuringExecution
软亲和力：尽量部署在满足条件的节点上，或尽量不要部署在被匹配的节点上

# 节点亲和力

```
      affinity:
        nodeAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
            nodeSelectorTerms:
            - matchExpressions:
              - key: kubernetes.io/os
                operator: In
                values:
                - linux
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 1
            preference:
              matchExpressions:
              - key: label-1
                operator: In  #NotIn pod就都走到node1里面了

                values:
                - key-1
          - weight: 50
            preference:
              matchExpressions:
              - key: label-2
                operator: In
                values:
                - key-2

```

kubectl label no k8s-node1 kubernetes.io/os=linux

kubectl label no k8s-node1 label-1=key-1
kubectl label no k8s-node2 label-2=key-2

# pod 亲和力

Pod 亲和力：将与指定 pod 亲和力相匹配的 pod 部署在同一节点。

Pod 反亲和力：根据策略尽量部署或不部署到一块

# s2-nginx-deploy-affinity.yaml

```
apiVersion: apps/v1 #deployment api 版本
kind: Deployment   #资源类型 deployment
metadata:  #元信息
  labels:  #标签
    app: nginx-deploy-s2  #key value
  name: nginx-deploy-s2    #deployment 的名字
  namespace: default  #所在的命名空间
spec:
  replicas: 1  #期望副本数
  revisionHistoryLimit: 10  # 进行滚动更新后保留的历史版本数
  selector:   #选择器,用于找到匹配的RS
    matchLabels: # 按照标签匹配
      app: nginx-deploy-s2    #匹配的标签 key value
  strategy:  #更新策略，
    rollingUpdate:   #滚动更新配置
      maxSurge: 25%   #进行滚动更新时，更新超过期望副本数多少个/比例
      maxUnavailable: 25% #进行滚动更新时，最大不可用的更更新比例
    type: RollingUpdate #更新类型，采用滚动更新
  template: #pod魔板
    metadata:  #元信息
      labels:
        security: S2
        topology.kubernetes.io/zone: V
        app: nginx-deploy-s2
    spec:  #pod期望信息
      affinity:
        nodeAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
            nodeSelectorTerms:
            - matchExpressions:
              - key: topology.kubernetes.io/zone
                operator: In
                values:
                - R
      containers: #容器
      - image: nginx:1.7.9 #镜像
        imagePullPolicy: IfNotPresent  #拉取策略
        name: nginx  #容器名称
        resources:
          limits:
            cpu: 200m
            memory: 100Mi
          requests:
            cpu: 100m
            memory: 100Mi
      restartPolicy: Always  #重启策略
      terminationGracePeriodSeconds: 30 #删除操作，最多宽限

```

# nginx-affinity-deploy.yaml

```
apiVersion: apps/v1
kind: Deployment
metadata:
  labels:
    app: nginx-deploy
  name: nginx-deploy
  namespace: default
spec:
  replicas: 1
  revisionHistoryLimit: 10
  selector:
    matchLabels:
      app: nginx-deploy
  strategy:
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 25%
    type: RollingUpdate
  template:
    metadata:
      labels:
        app: nginx-deploy
    spec:
      affinity:
        podAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
          - labelSelector:
              matchExpressions:
              - key: security
                operator: In
                values:
                - S1
            topologyKey: topology.kubernetes.io/zone
        podAntiAffinity:
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 100
            podAffinityTerm:  # 修正位置
              labelSelector:
                matchExpressions:
                - key: security
                  operator: In
                  values:
                  - S2
              topologyKey: topology.kubernetes.io/zone  # 添加 topologyKey 字段
      containers:
      - image: nginx:1.7.9
        imagePullPolicy: IfNotPresent
        name: nginx
        resources:
          limits:
            cpu: 200m
            memory: 100Mi
          requests:
            cpu: 100m
            memory: 100Mi
      restartPolicy: Always
      terminationGracePeriodSeconds: 30
```
