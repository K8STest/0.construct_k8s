# 创建一个 deployment

kubectl create deploy nginx-deploy --image=nginx:1.7.9

kubectl get deploy nginx-deploy -o yaml

```
apiVersion: apps/v1
kind: Deployment
metadata:
  annotations:
    deployment.kubernetes.io/revision: "1"
  creationTimestamp: "2024-03-17T22:48:37Z"
  generation: 1
  labels:
    app: nginx-deploy
  name: nginx-deploy
  namespace: default
  resourceVersion: "115484"
  uid: 73824838-a390-4b64-8c4b-2bdfd1f5551d
spec:
  progressDeadlineSeconds: 600
  replicas: 1
  revisionHistoryLimit: 10
  selector:
    matchLabels:
      app: nginx-deploy
  strategy:
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 25%
    type: RollingUpdate
  template:
    metadata:
      creationTimestamp: null
      labels:
        app: nginx-deploy
    spec:
      containers:
      - image: nginx:1.7.9
        imagePullPolicy: IfNotPresent
        name: nginx
        resources: {}
        terminationMessagePath: /dev/termination-log
        terminationMessagePolicy: File
      dnsPolicy: ClusterFirst
      restartPolicy: Always
      schedulerName: default-scheduler
      securityContext: {}
      terminationGracePeriodSeconds: 30

```

# 命令

只有修改了 deployment 配置文件中的 template 中的属性后，才会触发更新操作
kubectl get deploy,rs,po --show-labels
kubectl set image deployment/nginx-deploy nginx=nginx:1.9.1

### 回滚

kubectl set image deployment/nginx-deploy nginx=nginx:1.91 --record

#### 查看更新状态

kubectl rollout status deployments nginx-deploy

#### 查看有多少历史版本

kubectl rollout history deploy nginx-deploy

#### 查看某一个历史版本具体信息

kubectl rollout history deploy nginx-deploy --revision=3

#### 回退到某一个版本

kubectl rollout undo deployment nginx-deploy --to-revision=4
kubectl edit deploy nginx-deploy
kubectl get rs

需要指定保存版本个数为前提
.spec.revisonHistoryLimit 来指定 deployment 保留多少 revison，
