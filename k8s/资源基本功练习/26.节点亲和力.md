# NodeAffinity 介绍

节点亲和力：进行 pod 调度时，优先调度到符合条件的亲和力节点上

## RequiredDuringSchedulingIgnoredDuringExecution

硬亲和力，即支持必须部署在指定的节点上，也支持必须不部署在指定的节点上

## PreferredDuringSchedulingIgnoredDuringExecution

软亲和力：尽量部署在满足条件的节点上，或尽量不要部署在被匹配的节点上

# 编辑 nginx-deploy.yaml

```
apiVersion: apps/v1
kind: Deployment
metadata:
  labels:
    app: nginx-deploy
  name: nginx-deploy
  namespace: default
spec:
  replicas: 1
  revisionHistoryLimit: 10
  selector:
    matchLabels:
      app: nginx-deploy
  strategy:
    rollingUpdate:
      maxSurge: 25%
      maxUnavailable: 25%
    type: RollingUpdate
  template:
    metadata:
      labels:
        app: nginx-deploy
    spec:
      affinity:  # 添加亲和力配置
        nodeAffinity:
          requiredDuringSchedulingIgnoredDuringExecution:
            nodeSelectorTerms:
            - matchExpressions:
              - key: kubernetes.io/os
                operator: In
                values:
                - linux
          preferredDuringSchedulingIgnoredDuringExecution:
          - weight: 1
            preference:
              matchExpressions:
              - key: label-1
                operator: In
                values:
                - key-1
          - weight: 50
            preference:
              matchExpressions:
              - key: label-2
                operator: In
                values:
                - key-2
      initContainers:
        - image: nginx
          imagePullPolicy: IfNotPresent
          command: ["sh", "-c", "echo 'inited;' >> /.init"]
          name: init-test
      containers:
      - image: nginx:1.9.1
        imagePullPolicy: IfNotPresent
        name: nginx
        resources:
          limits:
            cpu: 200m
            memory: 100Mi
          requests:
            cpu: 100m
            memory: 100Mi
      restartPolicy: Always
      terminationGracePeriodSeconds: 30



```

# 命令

kubectl get no

实际上每一个节点都有了这个标签
kubectl label no k8s-node1 kubernetes.io/os=linux

kubectl get no --show-labels

kubectl label no k8s-node1 label-1=key-1
kubectl label no k8s-node1 label-2=key-2 #错误操作了
kubectl label node k8s-node1 label-2-

kubectl label no k8s-node2 label-2=key-2

删除 pod ，让重新调度
kubectl delete po nginx-deploy-7b96dd6897-dd567 nginx-deploy-7b96dd6897-rrd56 nginx-deploy-7b96dd6897-spj7p
kubectl get po -owide
预期都到节点二中部署 pod

# 修改 master 增加可用节点数

kubectl taint no k8s-master node-role.kubernetes.io/master:NoSchedule-
对于 master 来说 required 是满足的，
对于 prefered 是不满足的，但是也是一个可选的部署
此时如果修改 labe-1 的权重大一点，那就都往 k8s-node1 中部署了
