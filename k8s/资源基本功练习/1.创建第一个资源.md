# pod 探针的练习 httpGet

```
apiVersion: v1  # api文档版本
kind: Pod  # 资源对象类型 类似，deployment，statfulset
metadata:   # pod的相关元数据，用于描述pod 的数据
  name: nginx-demo-po  # pod的名称
  labels: # 定义pod的标签
    type: app  # 自定义label标签名字 为type，值为app，
    version: 1.0.0  # 自定义标签 描述的 pod版本号
  namespace: "default" # 命名空间的配置
spec:  # 期望pod 按照这里的描述进行配置
  containers: #对于pod中容器描述
  - name: nginx # 容器的名称，可随意指定
    image: nginx:1.7.9 # 指定容器的镜像 以及版本号
    imagePullPolicy: IfNotPresent  # 指定镜像拉取策略
    startupProbe:  # 启动探针
      httpGet:  #探测方式 基于http请求探测
        path: /index.html   #http请求路径
        port: 80 # 请求端口
      failureThreshold: 3 #失败多少次才算真正失败
      successThreshold: 1  # 成功的阈值
      timeoutSeconds: 5 #请求的超时时间
      periodSeconds: 10 # 间隔检测时间，比方说第一次检测失败了，再等10秒钟去检测
    command: #指定容器启动时 启动命令
    - nginx
    - -g
    - 'daemon off;' #nginx -g 'damemon off;'
    workingDir: /usr/share/nginx/html # 定义容器启动后的工作目录
    ports:
    - name: http # 端口名称
      containerPort: 80 # 描述容器内要暴露什么端口
      protocol: TCP # 该端口是基于哪种协议通通信的
    env: # 环境变量
    - name: JVM_OPTS # 名
      value: '-Xms128m -Xmx128m' # 值
    resources:
      requests: # 最少需要多少资源
        cpu: 100m #限制cpu 最少使用0.1个核心
        memory: 128Mi # 内存最少使用128M
      limits:  # 最多
        cpu: 200m
        memory: 256Mi
  restartPolicy: OnFailure # 只有失败 才会重启

```
