# 通过 edit 命令直接修改 configmap

```
kubectl edit cm test-dir-config

kubectl exec -it private-image-pull-pod -- sh

```

#

```
需首先修改test 文件夹下的文件内容，

kubectl create cm test-dir-config  --from-file=./test/ --dry-run -o yaml | kubectl replace -f-

```

# 不可变的 Secret 和 ConfigMap

```
对于一些敏感服务的配置文件，在线上有时是不允许修改的，此时在配置 configmap 时可以设置 immutable: true 来禁止修改

kubectl edit cm test-dir-config


```

```
apiVersion: v1
data:
  db.properties: |
    username=xiaoliu
    password=123456
  redis.properties: |+
    host: 127.0.0.1
    port: 6379

kind: ConfigMap
metadata:
  creationTimestamp: "2024-02-02T23:25:23Z"
  name: test-dir-config
  namespace: default
  resourceVersion: "165659"
  uid: f8d2bef1-e9d5-4f2c-9384-96bd5a7f84df
immutable: true
~
```
