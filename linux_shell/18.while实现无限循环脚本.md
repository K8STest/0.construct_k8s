大家好，我是博哥爱运维。前面课程有讲到过 for 循环，在一个有限条件下会一直迭代，直到满足条件退出，今天讲到的 while，我们可以利用它来实现一个阻塞式地无限循环脚本，那么这个有什么用呢，我们知道，在容器运行中，必须是有一个阻塞式地服务一直在前端运行，才能保证容器持续运行，而不被终止退出，那么这个时候，如果我们在容器里面实现一个 while 无限循环脚本，就可以保证这个容器一直运行，对于有时候我们需要运行一些运维工具的容器特别有帮助。

```
#!/bin/bash
while :;do   # : 是一个始终返回 true 的空命令，因此 while : 会创建一个无限循环。
    date
    sleep 1
done

while true;do
    date
    sleep 1
done

```

当然 while 也可以指定结果条件，使用方式和 for 循环基本类似:

```
#!/bin/bash

# 这个脚本使用 while 循环输出 1 到 10 的数字

counter=1

while [ $counter -le 10 ]; do
    echo $counter
    counter=$((counter + 1))
done




counter=1

while [[ $counter -le 10 ]]; do
    echo $counter
    counter=$((counter + 1))
done
```

[ ... ]：这是 POSIX 标准的测试命令，通常被称为 test。它适用于基本的条件检查，如数字比较、字符串比较、文件测试等。它是脚本中更常见的语法，具有更广泛的兼容性。
[[...]]：这是 Bash 特有的扩展测试命令，提供了更多的功能和灵活性。例如，它支持更复杂的条件表达式、模式匹配（如 =~ 正则表达式）、逻辑运算符 && 和 ||，并且在处理带有空格或特殊字符的字符串时更加安全。

seq 10：seq 是一个生成序列的命令，seq 10 会生成从 1 到 10 的数字序列
for i in `seq 10`; do echo ".... $i ....";done
for i in $(seq 10); do echo ".... $i ...."; done # 使用了 $() 而不是反引号来执行子命令 seq 10。$() 是一种更现代且推荐的方式来执行子命令
