这节课给大家带来这些 Linux 操作命令（diff, du, basename, dirname, df）

```
diff    文件内容对比命令，文件行数不多 能用

diff 1.log 2.log
3a4   # 第一个文件三行，第二个四号，a 增加
> 4



du      查看文件或目录使用空间大小

du -sh /root/
-s 选项告诉 du 命令仅显示总计，而不列出每个文件或目录的详细信息。
-h 选项告诉 du 命令以易于阅读的格式（如 K、M、G 等）显示文件大小。


basename   过滤出路径下面最后的文件名或目录， 取文件名字，或者文件夹名字
basename /opt/linux-shell/diff/1.log
basename /opt/linux-shell/diff

dirname    从文件名中删除最后一个组成部分
dirname /opt/linux-shell/diff/


df      查看系统硬盘占用空间情况
df -T
df -Th

-T: 显示文件系统的类型。
-h: 以人类易读的格式显示磁盘容量（例如，1K, 1M, 1G 等）




企业真实案例（面试中也会常常被问到）
    无法写入文件提示“no space left on device”  中文：设备上没有剩余空间
    用df -Th查看，分区使用空间并没有满，这个时候你会怎么去解决？



# df -i  查看 inode数是否满了


Filesystem      Inodes   IUsed   IFree IUse% Mounted on
/dev/xvda1     1310720 1310720       0  100% /
tmpfs          1007257       1 1007256    1% /dev/shm
/dev/xvdb1     5242880    3395 5239485    1% /logdata

查找：
du 结合 find /mnt -type f -size -100k|wc -l
解决：
由于小文件数量过多，直接在此目录下rm是会报错的，这里就用find形式来删除
find /mnt -type f -size -100k|xargs rm -f

find ./ -type f -size -3k # 低于3k的文件
find ./ -type f -size -5k| xargs rm -f

find ./ -type f -size +5k  # 找出大于5k的文件
```
