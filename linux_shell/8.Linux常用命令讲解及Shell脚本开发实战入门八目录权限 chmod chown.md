Linux文件目录权限
注：这个对于管理linux文件目录权限很关键，大家注意听，听完多操作，
理解好对后面操作linux系统运维非常有帮助

我们ls -l 一个文件 ， 或ls -ld 一个目录，通知在文件最前面rwx这些相关字符

touch 1.log
chmod +x 1.log

文件：
# ls -l 1.log
-rw-r--r-- 1 root root 0 Sep  6 21:47 1.log
目录：
# ls -ld dir
-rw-r--r-- 1 root root 0 Sep  6 21:48 dir


r   w   x
4   2   1
读  写  执行or进入
0644/-rw-r--r--  # 文件
0755/drwxr-xr-x  # 目录

rwxr-xr-x 表示权限信息，分为三组：
第一组 rwx 表示文件所有者的权限，这里拥有者可以读、写和执行。
第二组 r-x 表示所属组的权限，组成员可以读和执行，但不能写。
第三组 r-x 表示其他用户的权限，同样可以读和执行，但不能写


chmod 777 1.log  # 最大权限  rwx是一组

ll -d dir # 目录

umask
    0022 # 第一位0: 表示这是一个八进制数

chown   更改文件的所有者及所有组,常用参数-R 将目录内所有目录及文件都作修改  chown abel.abel 1.log
chown abel.abel -R ./dir

chmod   更改文件的权限
  chmod +x 文件
  chmod 777 目录或文件



lsattr  显示文件特殊权限操作
chattr +i # 锁定文件，只读  chattr -i #去掉
chattr +a # 只允许增加内容  chattr -a #去掉
