大家好，我是博哥爱运维。这节课给大家带来的是利用 shell 脚本，来实现并发查询一个网段内，所有在线的 IP，并将在线和未在线的 IP 地址进行分类保存。

掌握内容：

1、ping 命令的使用

2、shell 脚本里面的并发编程

3、状态文件保存

并发特性：

1、根据 for 循环的条件，生成多个子进程来实现并发任务。

2、并发执行的最后，wait 会等所有子进程返回结果后再退出主进程脚本。

源码如下：

```

#!/bin/bash
#__author__: boge

> ./ip_offline.txt  # 创建一个名为 ip_offline.txt 的文件（如果文件已存在，则清空其内容）
> ./ip_online.txt

if [ $# -ne 3 ];then
	echo "Usage: bash $0 192.168.1 1 254" # 网络位，主机起始位和结束位
	exit 3
fi

pingnum=`seq $2 $3`
# $(seq 1 5)
# 1 2 3 4 5

for i in $pingnum
do
{
  ping -c 1 -i 0.1 -W 1 $1.$i &> /dev/null
  if [ $? -eq 0 ];then
    echo "IP:   $1.$i is online."
    echo "$1.$i" >> ./ip_online.txt
  else
    echo "IP $1.$i is offline."|tee -a ./ip_offline.txt

     # 标准输出: tee 默认会将从标准输入读取的内容打印到标准输出（通常是终端）。
     # 文件输出: 如果提供了文件名，tee 也会将相同的内容写入该文件
     # -a 选项告诉 tee 将输出追加到指定文件的末尾，而不是覆盖文件。这对于在现有文件中添加更多内容非常有用。
  fi
}&  # & 每一次迭代操作 ，都后台执行
# { ... } &:
# 使用 { ... } & 将循环体放入后台执行，这样可以并行执行多个 ping 命令。

done
wait  # 等待所有子进程结束

```

# 操作

ip r

ping 192.168.1.25
PING 192.168.1.25 (192.168.1.25) 56(84) bytes of data.
64 bytes from 192.168.1.25: icmp_seq=1 ttl=64 time=0.020 ms
64 bytes from 192.168.1.25: icmp_seq=2 ttl=64 time=0.025 ms

ttl=64：表示数据包的生存时间（Time To Live）。每经过一台路由器，TTL 值会减一，初始值通常是 64，所以这个值显示 64 表明目标主机与源主机在同一子网内

ping -c 1 192.168.1.25

ping -c 1 -i 0.1 192.168.1.25 # 用于向 IP 地址 192.168.1.25 发送一个 ICMP 请求包，并限制发送次数为 1 次，同时设置两次连续 ICMP 请求之间的间隔时间为 0.1 秒

ping -c 1 -i 0.1 -w 1 192.168.1.26
time ping -c 1 -i 0.1 192.168.1.26

```
ping: 用于测试网络连通性。
-c 1: 限制发送 ICMP 请求的次数为 1 次。
-i 0.1: 设置两次连续 ICMP 请求之间的间隔时间为 0.1 秒（虽然在这种情况下只发送一次，因此实际影响不大）。
-w 1: 设置等待响应的超时时间为 1 秒。
```
